Technical Deep-Dive: Draft Graph Mode

Problem

Draft model in eager mode: each of 3 Markov steps re-launches NPU kernels. 30ms overhead (23% of verify_time). Previous attempt (Step 3) using ACLGraphWrapper failed — custom _propose bypassed runner dispatch pipeline, triggering vllm safety lock.

Solution: Direct torch.npu.graph in _propose

# In _propose (deepseek_v4_dspark_proposer.py):
self._precompute_context_kv()  # Eager: update KV cache pool

with set_ascend_forward_context(None, ...):
    fc = get_forward_context()
    if fc: fc.moe_layer_index = 0

    if num_tokens in self._draft_graphs:
        # REPLAY PATH (subsequent calls)
        self._draft_graphs[num_tokens].replay()
        hidden = self._draft_graph_outputs[num_tokens][:n].clone()

    elif self._use_draft_graph:
        # CAPTURE PATH (first call)
        set_cudagraph_capturing_enabled(True)
        for _ in range(3):          # warmup
            md = self._build_attn_metadata(cad)
            _ = self.model(input_ids=..., positions=...)
        torch.npu.synchronize()

        graph = torch.npu.NPUGraph()
        with torch.npu.graph(graph, pool=global_pool):
            # KEY: _build_attn_metadata INSIDE graph context
            # → cos/sin tensors in graph pool (stable addresses)
            md = self._build_attn_metadata(cad)
            output = self.model(input_ids=..., positions=...)
        self._draft_graphs[num_tokens] = graph
        set_cudagraph_capturing_enabled(False)

    return self._sample_sequential(hidden, ...)  # Markov eager

Why It Works

ChallengeSolution
cos/sin tensors created outside graph → addresses unstable on replay_build_attn_metadata called inside torch.npu.graph() — tensors allocated from graph pool, addresses persist
query_start_loc / seq_lens newly created each callPre-allocated buffers (_query_start_loc_buffer, _seq_lens_buffer), copy-in before each replay
vllm safety lock (cudagraph_capturing_enabled=False)Manually set_cudagraph_capturing_enabled(True) before capture, restore after
dispatch system bypass (no batch_descriptor)Not needed — torch.npu.graph() doesn't require vllm dispatch context
Compute_logits may overwrite graph output.clone() graph output before _sample_sequential
