局限性说明

  • 硬件维度:清单 A2/A3 为合并列,无法区分 A2 vs A3,记为 A2A3 桶;Ascend 950 列记为 A5。废弃提示行缺硬件列时记 unknown。
  • 场景维度:op_api_list.md 不区分 inference/training,scenario 一律 unknown;训练支持需看 aclnnXxxGrad 反向变体。
  • 覆盖范围:仅 ops-math/ops-nn/ops-transformer。op-plugin(7.3.0 分支无 9.0.x tag)、ATB(9.0.1 沿用 9.0.0 分支)、torch_npu(版本不变)未纳入。
  • watchlist 命名:已用 DSV4_ALIASES 修正 trace table 的大小写/后缀误写(aclnnGatherPaKVCache→aclnnGatherPaKvCache、aclnnScatterNdUpdateV2→aclnnScatterNdUpdate)。
  • 计数方法学:每行按首个链接归属;描述中跨仓引用链接(如 ops-transformer 第 197 行描述里的 [aclnnRmsNorm](...ops-nn...))不计为本仓 API,故 ops-transformer 计 156/169(朴素 grep 会多算至 157/170)。aclnnRmsNorm 实属 ops-nn。
  • 上游重复:aclnnMlaPrologV2WeightNz 在 ops-transformer 清单两版本各出现 2 次(上游重复),inventory 保留两行作证据,diff 为集合运算不受影响。
  • 证据:每条结论可经 git -C submodules/<repo> show <tag>:docs/zh/op_api_list.md + 行号复现。
