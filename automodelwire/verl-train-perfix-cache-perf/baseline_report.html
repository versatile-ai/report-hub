⑧ 结论与下一步

基线特征
  • update_actor 是首要瓶颈 (282s/步, 45%)，其中反向 175s 占 actor 64% → 前缀缓存优化若能减少 actor 前向/反向 token 量，收益最大
  • gen (rollout) 波动大 (125–147s)，受 response_length (1136–1232) 与 vLLM 调度影响
  • old_log_prob / ref 高度稳定 (~100s)，是前缀合并/复用的理想靶点 (两阶段对同 batch 重复前向)
  • 8 卡负载极均衡，无 straggler；HBM 余量充足 (峰值 29.5/64GB)
  • MFU 仅 4.27%，计算密度低，属于 RL 训练典型 (小 batch + 长序列 + 多次前向)
下一步
  • 当前为 3 步 smoke，需跑满 warmup10 + collect100 得稳态统计
  • 依次铺开 G3 (前缀合并) / G4 (合并+prefix-sharing) 对比 G1 基线，验证 old_log_prob+ref 阶段加速
  • 本目录未用 workflow template，建议规整为标准 5 件套结构
