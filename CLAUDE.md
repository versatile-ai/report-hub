# report-hub 模块说明

## 接口边界

本模块是 versatile-ai 的**静态 HTML 报告聚合站点**,只承载自包含 HTML 文件(含内联 CSS/JS,无外部依赖、无构建步骤)。不包含任何后端逻辑、数据库、动态渲染服务。

对外接口仅为 HTTP 静态文件服务:
+ 顶层 `index.html` = 仓库级聚合导航(versatile-ai 全局入口)
+ `子目录/index.html` = 各仓库的仓库级导航
+ `子目录/.../*.html` = 具体报告页(自包含,可直链分享)

## 功能

1. **仓库级聚合**:顶层 `index.html` 通过 `REPOS` 数组列出所有仓库,每项指向子目录导航页。
2. **仓库级导航**:每个仓库子目录有自己的 `index.html`,负责列出本仓库的实验与报告(数据由各仓库自行维护)。
3. **直链分享**:每份报告是自包含 HTML,URL 即路径,可直接分享。

## 依赖关系

- **上游**:各仓库的 `gh-pages` 分支或其他静态产出,迁入对应子目录。
- **下游**:浏览器直接渲染,无服务端依赖。
- **构建**:无。`python3 -m http.server` 或任意静态服务器即可预览。

## 设计规范

全站视觉系统以 `docs/design-guide.md` 为单一事实源:主题基调(浅壳入口页 + 深文报告页)、
design tokens(颜色浅/深两态、字体三角色、间距、圆角)、组件规格、报告页骨架模板、
可访问性要求。新增或修改任何页面前先读该文档,产物须自包含内联 CSS(gh-pages 约束)。

## 新增仓库的约定

1. 子目录名 = GitHub 仓库名(全小写,连字符分隔)。
2. 子目录内必须有 `index.html` 作为仓库级导航。
3. 报告文件保留仓库内相对路径,确保导航链接不破坏。
4. 在顶层 `index.html` 的 `REPOS` 数组登记新仓库,字段仅限 `id/name/path/gh/desc/lastUpdate/tags`。
5. **聚合统计不复制**:实验数、报告数等由各仓库子目录 `index.html` 自行维护(其 `EXPERIMENTS`/`REPOS` 等数组即事实源)。顶层 `REPOS` 不持有 `reports`/`active`/`stats` 等聚合字段,顶层 stats ribbon 只报"仓库数 / 最近一次仓库级更新"两个顶层确实掌握的事实,避免顶层与仓库页双源漂移。
